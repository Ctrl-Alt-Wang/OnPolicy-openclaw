# 文献状态标注规范

借鉴成熟实证论文 skill 的「结果三态」(planned/placeholder/verified) 理念，
改造为基金撰写场景的**文献三态**。核心目的：**绝不编造文献**，让每条引用的可信度
透明、可追溯。

## 三态定义

| 状态 | 含义 | 文内表现 |
| --- | --- | --- |
| `pending` | 待检索：尚未调用文献检索工具 | 引用位点尚未布设 |
| `placeholder` | 占位：已布设引用位点 `[n]`，文献条目待填充 | 文内有 `[n]`，文末列表为空/占位 |
| `verified` | 已核实：引用来自检索工具真实返回并已核对 | 文内 `[n]` ↔ 文末真实文献一一对应 |

## 不可协商的铁律

1. **绝不编造**任何文献条目（标题、作者、期刊、年份、PMID/DOI 一律不得虚构）。
2. 不得把 `placeholder` 谎标为 `verified`。
3. 任何 `Citation_Need=yes` 的模块，在 `Citation_Status` 达到 `verified` 之前，
   对应 issue **不得标 `Status=DONE`**。
4. 文献检索工具不可用时：如实保留 `placeholder`，在 Notes 注明原因，
   并在交付时明确告知用户「引用待接入文献库后填充」。

## 标准检索调用方式（工具无关描述）

> 在 InfoX-Med 环境中即调用内部医学文献检索 skill / API。本 skill 不绑定具体接口，
> 仅约定输入输出语义，便于部署到任意模型/系统。

**输入**：检索词（中英文均可，可含布尔逻辑与 MeSH 词），时间范围默认近 10 年。

**期望返回**：每条文献至少包含 标题 / 作者 / 期刊 / 年份 / 唯一标识（PMID 或 DOI），
按相关度或年份排序。

**写作中的使用顺序**：
1. 根据模块主题构造检索词（见各模块 specs 的「前置操作」）。
2. 调用检索工具，拿到近 10 年真实文献。
3. 据此撰写，文内布设 `[n]`，文末统一编号列表。
4. 将该 issue 的 `Citation_Status` 更新为 `verified`。

## 引用格式

- 全文统一：文内数字顺序引用 `[n]`，文末按出现顺序编号列出。
- 参考文献仅在**全文结尾输出一次**（对应 issue `Z1`）。
- 格式风格全文一致（建议国自然常用的数字顺序制）。
