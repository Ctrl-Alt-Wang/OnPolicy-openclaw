# 修改和优化（三、修改和优化）

撰写完成后，用户可对已生成的任意正文段落进行修改和优化。本阶段不是重写，而是
在保留原意与结构的前提下，按既定质量维度打磨选中的内容。

## 工作方式

- 相当于一个「在线编辑器」：用户选中某段已生成的正文，提出修改诉求。
- 针对用户**选中的那一段**进行编辑优化，不擅自改动未选中的其他部分。
- 修改后应保持与全文其他模块的衔接、术语与引用一致。

## 优化的关键维度

对选中内容从以下维度检查并提升：

| 维度 | 含义 |
| --- | --- |
| 准确性 | 事实、数据、文献引用准确无误 |
| 规范性 | 符合国自然申请书的行文与格式规范 |
| 连贯性 | 段落内部及与上下文的逻辑衔接顺畅 |
| 透明性 | 推理与论证可追溯，结论有依据 |
| 一致性 | 术语、口径、引用风格与全文统一 |
| 深度优化 | 论证更充分、表达更专业，避免空泛 |

## 操作要点

1. 先识别用户诉求属于哪几个维度（可多选）。
2. 仅在该段落范围内修改；如发现需联动其他模块，先提示用户，不擅自改动。
3. 修改不得违反原模块的规格约束（字数范围、引用要求、是否分点等，见
   `module-specs.md`）。
4. 涉及文献/数据的修改，仍需以文献检索工具的真实结果为依据。
